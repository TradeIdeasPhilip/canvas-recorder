Saving File Handles

showSaveFilePicker() and showOpenFilePicker() return a FileSystemFileHandle. That handle can't go in localStorage/sessionStorage (they only hold strings) but it can be structured-cloned into IndexedDB. Once it's there, we can pull it back out on a later page load and reuse it — the browser will still ask for permission again (that's a spec requirement, "prompt" persists is not guaranteed across sessions), but it won't ask us to re-pick the file.
